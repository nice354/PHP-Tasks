## Задание 1 — Что мы видим? Разбор интерфейса VK

Я выбрал приложение **VK**, экран — лента новостей.

Вот как выглядит экран и что на нём происходит:


┌──────────────────────────────┐
│  VK         🔍        🔔     │   ← статичная шапка (HTML)
├──────────────────────────────┤
│  [😊] Что у вас нового?      │   ← форма, всегда на месте (HTML)
├──────────────────────────────┤
│  [🧑] Артём Иванов           │   ← имя и аватар — с сервера (JS)
│  Сегодня был крутой день...   │
│  ❤️ 24   💬 3   ↗️ Поделиться│   ← счётчики меняются (JS)
├──────────────────────────────┤
│  [🧑] Маша Петрова           │   ← следующий пост (JS)
│  Смотрите какой закат 🌅     │
│  ❤️ 87   💬 12               │
├──────────────────────────────┤
│  🏠    👥    💬    🔔    👤  │   ← нижнее меню, не меняется (HTML)
└──────────────────────────────┘


**Что я отметил:**

- HTML (структура): шапка с логотипом, нижнее меню, поле «Что у вас нового?» — всё это стоит на месте и не меняется
- CSS (оформление): фирменный синий цвет VK, скруглённые аватарки, отступы между постами, плавная анимация при нажатии на лайк
- JavaScript (интерактив): лайк меняет цвет и счётчик сразу, лента подгружается при скролле, уведомления появляются без перезагрузки страницы

**Вывод:** фронтенд — это всё, что я вижу и с чем взаимодействую прямо на экране. Он отображает данные и реагирует на мои действия, но сам данные не хранит.

## Задание 2 — Эксперимент: отключаем интернет

Включил режим «В самолёте» прямо в ленте VK.

| Что проверял | Результат |
| Что осталось на экране? | Старые посты, которые уже были загружены — остались |
| Что перестало работать? | Новые посты не грузятся, лайк не ставится, сообщение не отправляется |
| Какие ошибки? | Появилась плашка «Нет подключения», кнопки стали неактивными |

После включения интернета сделал pull-to-refresh:

1. Сначала появился крутящийся индикатор — это **фронтенд** (JS) показывает, что идёт загрузка
2. Через секунду появились новые посты — это уже результат работы **бэкенда**, который вернул данные

**Вывод:** без интернета фронтенд может показать только то, что уже закэшировано. Всё живое и актуальное — приходит с сервера.

## Задание 5 — Моя идея: приложение «Как ты сегодня?»

Приложение для отслеживания настроения. Каждый день отмечаешь, как себя чувствуешь, а оно строит статистику и присылает что-нибудь приятное.

### Что делает фронтенд:
- Три большие кнопки: Хорошо / Нормально / Плохо
- Календарь, где каждый день закрашен своим цветом
- График настроения за последние 30 дней
- Карточка с мотивирующей цитатой (приходит с сервера)

### Что делает бэкенд:
- Хранит всю историю настроений пользователя
- Анализирует закономерности (например, по понедельникам чаще плохое настроение)
- Подбирает цитату под текущее состояние
- Считает среднее настроение за неделю/месяц

### Схема общения:


  ФРОНТЕНД                                        БЭКЕНД
  ────────                                        ───────

  Пользователь нажал 
        │
        ▼
  POST /mood  ──────────────────────────────────►  принимает запрос
  {                                                 сохраняет в БД
    "mood": "good",                                 подбирает цитату
    "date": "2026-03-27",                           считает streak
    "note": "сдал экзамен"
  }
        
        ◄──────────────────────────────────────── возвращает ответ
  {
    "status": "saved",
    "streak": 5,
    "quote": "Продолжай в том же духе!",
    "avg_week": "good"
  }
        │
        ▼
  Показывает: «Настроение сохранено»
  + цитата дня + «5 дней подряд — огонь»


**Запрос статистики за месяц:**
json:
GET /stats?period=2026-03

Ответ:
{
  "good": 14,
  "neutral": 9,
  "bad": 4,
  "best_day": "пятница",
  "worst_day": "понедельник"
}
